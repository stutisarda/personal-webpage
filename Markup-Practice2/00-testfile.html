Inline (Phrasing) Elements

Inline elements are used inline within a block element, such a paragraph. For example, if you want some text to have extra weight, you might use a strong tag. Or if you want to emphasize text you might use an em tag for emphasis. The way these tags render by default is that strong makes text bold, and em makes text italic in the browser, but you can define how you want those to look with css.

HTML5 also resuscitates the old for bold and for italic tags. However these are used when you want to mark content for a specific meaning other than for strength or emphasis. For example, you might use the i tag for the title of a magazine article in a bibliography. In that case you are not trying to emphasize the title of the magazine, but rather format the content in a particular way.

Another inline, phrasing element is the anchor tag. The anchor tag is the glue that holds the World Wide Web together. The anchor tag is how you link to other pages. It looks like this: link to Google Notice the href attribute. Like all attributes it has the name of the attribute followed by an equal sign, followed by the value of the attribute in quotes. This attribute gives the tag more information. What am I linking to? I am linking to the Google website.

Generic Tags and HTML5 tags

I mentioned inline tags earlier. All tags either display as inline (phrasing) elements or block (structural) elements. By block elements I mean by default, the browser will create a new block on the page, with some amount of space before the block, and some amount of space after for what ever content comes before or after. How much space can be defined with CSS. Some of the block elements we have looked at already are the P tag, the H tags and the unordered and ordered list tags, as well as the list item tags. The table tag is a block element.

Inline elements display inline with the res of the page. For example, the strong tag will make text strong inline with a paragraph. The anchor tag is an inline element by default. So are images, which is often problematic. You can make an inline element display like a block element and vice versa with CSS. We will do that frequently, as you will see later, but for now, it is important to know which is which by default.

There are generic block and inline elements, used for layout purposes. They are div and span respectively, and look like this...

This is a generic division for a document. It will make a generic block element.

Here is a div block with a paragraph block inside it. This paragraph has a span inside it. The span tag is a generic inline element and can be used in combination with css to define a look for anything between the span tags. You probably won't use span a lot, but you will use the div tag from time to time. You use the div tag less now because there are some nice HTML5 block level tags that replace a lot of div tags that we used to use. Here are some you might see...

Header is a generic block element that is used to describe the header of a page or article or section. You would use it in place of a div tag. The word, "header" has meaning and helps us understand that the content inside the header has some importance as the beginning of something.
The section tag is used to denote a section of a document. A section could have other blocks inside of it, or it could be inside of other blocks.
Article is another block element that helps give our document definition. An article could have sections, or a section could have articles. Think about a newspaper. Perhaps you are in the sports section, and there are several articles. Or perhaps you are reading a long article, and it is broken into different sections. The important thing here is to think about your content and how it breaks down structurally.
The aside element is another block element used to describe separate but related content, like a sidebar.

You will notice that if you look at this page in a web browser, you can not see the difference between any of these different block elements. The point is that they add structure to your document so that Google and other devices can make sense of them. This helps search engines understand your pages and index them properly. It helps when you need to edit the page later. Part of our job is to give our documents a logical structure so that they make sense. With CSS we can define any of these elements to look any way we want, and we will start doing that soon.

This is not all of the elements, but this is a good chunk of them, and it is the ones you will use most frequently.
